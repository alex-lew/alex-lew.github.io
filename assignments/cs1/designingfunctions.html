Designing functions

Assignment overview

In this assignment, you’ll get some practice writing functions–in other words, teaching the computer new operations. Remember the process for writing a new function:

  1. Decide on a descriptive name for your function.

  2. Write a “contract” for your function, which describes the types of its arguments and the type of its output:

    1
    ; funcname : ArgumentType1 ArgumentType2 ... ArgumentTypeN -> ResultType

    The types currently available to you are Number, String, Image, and Boolean.

  3. Write a “purpose statement” for your function, which is the simplest possible answer to “what does this function compute?”

  4. Write examples, if you’d like:

    1
2
3
4
    ; area-of-square : Number -> Number
; Computes the area of a square with given side length
; (area-of-square 5) should give 25
; (area-of-square 7) should give 49

  5. Based on your concrete examples, choose descriptive names for your argument variables, and extract a function definition:

    1
2
    (define (area-of-square side-length)
  (* side-length side-length))

Functions

Please design the following functions, in a file called YourNameDesigningFunctions.rkt. At the end of this document, I’ve provided a list of potentially useful built-in functions.

1Area of an image

Design a function that takes in a single argument, an image, and computes its area in pixels.

2Distance from origin

Design a function that takes in two arguments, x and y coordinates of a point, and calculates the distance between (0, 0) and (x, y). (This is a Geometry question: use the distance formula!)

3Framing an image

Design a function that takes in one argument, an image, and returns that image “in a frame”–that is, placed in the center of an empty scene that is twice as big as the original image (in both dimensions).

4Birthday greeting

Design a function that takes in a person’s name, a string, and returns a birthday greeting for that person (e.g. “Happy birthday, Voldemort!“).

Extra Challenge: Country flags

Design a function that takes in a height, and produces an image of that height depicting the flag of South Korea, using only functions in the 2htdp/image.rkt teachpack. (That is, do not just copy and paste the flag into your program; create it using circles, rectangles, and rotations.) The flag looks like this:

Extra Challenge: Throwing a ball

(Requires some physics.) Design a function that takes in two numbers: the first is the number of seconds elapsed since a ball was thrown, and the second is a “gravity” constant (in pixels per seconds squared).

Your function should produce an image of a ball that has been flying through the air–in a parabolic shape, as balls do–for that number of seconds, in a world with that amount of gravity. Assume the ball is thrown at time 0 with velocity 5 pixels per second at an angle of 45 degrees. Its starting position is the bottom left of a 500 by 500 empty scene.

Extra Challenge: Clock

(May require some trigonometry.) Design a function that takes in an hour of the day, between 0 and 11, and minutes, between 0 and 59, and produces an image of a “clock” with an hour hand pointing in the correct direction for that hour and a minute hand pointing in the correct direction for that minute. For an extra extra challenge, make it so the hour hand’s location depends also on the minutes.

Useful built-in functions

You may find the following functions useful in completing this assignment:

  • image-width : Image -> Number
    Computes the width of an image, in pixels.

  • image-height : Image -> Number
    Computes the height of an image, in pixels.

  • sqrt : Number -> Number
    Computes the square root of a number.

  • sqr : Number -> Number
    Squares a number (multiplies it by itself).

  • empty-scene : Number Number -> Image
    Creates an empty rectangular scene of the given width and height, in which to place other images.

  • place-image : Image Number Number Image -> Image
    Places the first image on top of the second image to obtain a new image. The image is placed so that its center appears x pixels over and y pixels down from the top left corner of the screen, where x and y are the two numerical arguments.

  • overlay : Image Image -> Image
    Places the first image on top of the second image, so that the centers of each image are aligned. (That is, the first image is placed in exactly the center of the second.)

  • string-append : String String -> String
    Adds two strings together into one string, by concatenating (“gluing together”) the two arguments.
