Using type declarations

Assignment overview

In this assignment, you will practice writing type declarations and functions by creating several simple but (arguably) useful types based on ints and strings. Each problem will ask you to create a type with some functions that work with that type. You must also write a main function, which creates at least one value of each type and tests its functions.

Problems

1Name

Create a type name, with underlying type string. Also write the following functions:

  • greet(n name), which returns a string greeting of your choosing to the person with the given name.

  • opening(n name, salutation string), which creates a letter opening to someone with this name, using the given salutation.

  • closing(n name, signoff string), which creates a letter closing from someone with this name, using the given signoff.

2Christmas Tree

Create a type christmasTree, with underlying type int, representing its height. Write a draw function that causes a christmas tree of the given height, made out of # and | characters, to the Terminal. For instance, draw(christmasTree(4)) would print

   #
  ###
 #####
#######
  |||

Note: the number of # characters is always odd, and increases from top to bottom of tree. The trunk is always the same height.

Extra Challenge: ISBN

ISBN numbers uniquely identify books. So that it’s easy to check that an ISBN number is valid, and hasn’t been corrupted somehow (e.g., someone accidentally copying a digit wrong), real ISBN numbers all have the property that if you add up the digits, multiplying each by the index of the digit you’re on, you get a multiple of 11. For a fuller explanation, see the Wikipedia page (click here).

Design a type isbn, backed by either string or int (your choice). It should have two functions:

  • printIsbn(id isbn), which prints the ten-digit ISBN number out.

  • isValid(id isbn), which returns a boolean representing whether the number is valid or not.

Note: if you choose to use an int, you should know that in Go, typing a number with leading 0s causes it to be interpreted in base 8. So if your ISBN number begins with 0, you still encode it by writing isbn(...), where ... has no leading 0s.
